# Model Card: CardioRisk Predictor

## Model Details
- **Model Name:** CardioRisk Classification Model (Best Model: Gradient Boosting Classifier)
- **Version:** 1.0
- **Type:** Binary Classification Machine Learning Model
- **Developer:** Portfolio Project (Educational)
- **Date:** 2026-05

## Intended Use
- **Intended Use:** This model is intended for educational and analytics purposes to demonstrate the end-to-end data science process (data cleaning, modeling, threshold tuning, and explainability). It is meant to be a decision support demonstration to identify population-level risk segments.
- **Not Intended Use:** **This model is not clinically validated. It should not be used for diagnosis, treatment decisions, or replacing medical professionals.** It is intended only as a data science and analytics demonstration.

## Dataset
- **Name:** Cardiovascular Disease dataset
- **Source:** Kaggle (originally collected at the moment of medical examination)
- **Size:** 70,000 patient records (before cleaning)
- **Features Used:** `age_years`, `height`, `weight`, `gender`, `ap_hi`, `ap_lo`, `cholesterol`, `gluc`, `smoke`, `alco`, `active`, `BMI`, `pulse_pressure`, and engineered categories.
- **Target Variable:** `cardio` (1 = Presence of cardiovascular disease, 0 = Absence)

## Evaluation Metrics
The model is evaluated using the following metrics, with a strong emphasis on Recall/Sensitivity to minimize false negatives:
- Accuracy
- Precision
- Recall (Sensitivity)
- Specificity
- F1-Score
- ROC-AUC

## Best Model Performance
The best performing model (typically Gradient Boosting or Random Forest) achieves approximately:
- **Accuracy:** ~73.5%
- **ROC-AUC:** ~0.80
- **Recall (at default threshold):** ~70%
*(Note: These metrics vary slightly based on train/test split and hyperparameter tuning. Check the `reports/model_results.csv` for exact numbers generated by the pipeline.)*
Through threshold tuning, we demonstrated the ability to increase Recall (e.g., catching more high-risk patients) at the cost of increasing False Positives (lower Specificity/Precision).

## Important Features
Based on feature importance and permutation importance analyses, the top risk drivers identified by the model are:
1. Systolic Blood Pressure (`ap_hi`)
2. Age (`age_years`)
3. Cholesterol (`cholesterol`)
4. BMI (`BMI`) / Weight

## Ethical Considerations
- **Use Case:** Using predictive models in healthcare carries significant ethical weight. Predictions should augment, not replace, physician expertise.
- **Transparency:** The model's decision-making process is analyzed using feature importance methods to ensure the key drivers align with established medical knowledge (e.g., blood pressure and age being key risks).

## Limitations
- **Data Quality:** The original dataset contains impossible values (e.g., negative blood pressures, extreme heights/weights) that required aggressive heuristic cleaning. The surviving dataset might be biased by this cleaning process.
- **Missing Features:** Crucial medical data like family history, stress levels, resting heart rate, and detailed lab work (e.g., exact LDL/HDL numbers) are absent, limiting the model's predictive power.

## Bias Considerations
- The dataset does not specify geographic, socioeconomic, or racial demographics. Therefore, it is impossible to assess whether the model performs equitably across different marginalized groups. Applying this model to a demographic significantly different from the training population would likely result in degraded performance.

## Future Improvements
- Train the model on more comprehensive EHR (Electronic Health Record) data.
- Explore fairness-aware machine learning techniques to ensure predictions do not systematically bias against certain genders or ages.
- Implement robust local explainability (e.g., patient-level SHAP values) in the Streamlit application.
